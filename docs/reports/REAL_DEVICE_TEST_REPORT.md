# A80 真实设备测试报告

测试日期：2026-07-16 至 2026-07-17

测试对象：A80 手表（Android 8.1 / API 27，320 x 380）、MQTT 中继、HTTP 桥接服务和 Web 大屏

应用版本：`com.heatstress.watch` / `1.1.0-a80`（versionCode 2）

## 结论

本次在真实 A80 上完成了安装、开机自启动、前台采集服务、MQTT 在线状态、断线缓存补传、中文预警下发和
320 x 380 界面回归。手表主界面已恢复六月版的黑底、四仪表和高对比状态配色，同时保留当前版本的全部数据与
告警行为。桥接服务的单元测试、远端部署、MQTT 校时和 Web 端空值回归也已通过。手表未佩戴时，
应用不会再把零值或历史测试值伪装成实时生命体征，大屏会将其归入“数据不可用”而不是“正常”。

当前结果证明 USB 联调链路可工作，但还不能视为生产现场验收通过。手表尚未配置独立网络；当前时间校正依赖
连接中继后的 MQTT 消息，冷启动离线阶段仍没有可信时间源。项目也没有完成佩戴采集、GPS 室外定位和长时间
续航测试；现有 Django 大屏后端还存在空健康值写入失败及历史值残留问题。

## 尚未完成的生产验收

1. **佩戴采集**：测试期间手表未实际佩戴，因此真实心率、血氧、血压、体温估算的准确性、连续性及四仪表同时显示真实值的场景尚未验证。
2. **独立网络**：当前 A80 无可用 Wi-Fi 或 SIM，MQTT 依赖 USB `adb reverse`。脱离电脑后无法连接中继。
3. **离线可信时间**：设备重启后时钟会回到 2012 年。连接 MQTT 后 bridge 已能校时，但首次建链之前和中继不可达时仍缺少可信 RTC/NTP；当前来源标记也必须配合 MQTT ACL/TLS 才能形成身份认证。
4. **GPS**：室内无定位结果，尚未完成室外首次定位、漂移、断网和重启回归。
5. **大屏后端空值语义**：后端接收显式 null 健康字段时返回 500；省略字段又会保留历史测试值。修复前，大屏上的旧生命体征不能视为本次实测值。
6. **MQTT 安全**：现网明文端口允许匿名访问，TLS 证书也未建立可信链。生产部署需要关闭匿名访问、使用设备级凭据并启用可信 TLS。
7. **耐久性**：尚未执行 8 小时以上续航、后台保活、弱网漫游、积压上限和服务器故障恢复测试。

## 验收建议

在现场验收前，先为手表配置独立网络与离线 RTC/NTP 兜底，并修复 Django 大屏后端的可空字段更新语义。随后至少完成一次
真人佩戴的完整班次测试，记录原始传感器帧、手表展示值、MQTT 消息、后端落库值和大屏展示值，以同一时间轴
核对全链路一致性。测试告警必须使用独立编号并标注为测试数据，避免与真实热应激事件混淆。
